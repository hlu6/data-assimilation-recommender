Dynamic Recommendation System

Adaptive recommendations for fast-changing user behavior.

PulseRank AI is a real-time recommendation concept that combines a temporal graph neural network with an online filtering step, allowing the system to both predict user interest and react to new interactions as they arrive. Instead of relying on a purely model-driven update rule, the system treats recommendation as a dynamic prediction-correction process: the graph model provides a structured prior, new interactions directly correct the current state, and the resulting loop supports more adaptive personalization in fast-changing environments.

How the framework differs from purely model-driven recommenders

Typical recommender behavior

-
Slower update cycle

New interactions often influence rankings only after more training or delayed model updates.

-
Staler state over time

Embedding states can lag behind fast-changing user behavior in streaming settings.

-
Limited direct correction

Observations are usually interpreted only through the learned model rather than updating the state immediately.

What this framework adds

+
Faster adaptation

Filtering shortens the gap between a new interaction and an updated recommendation state.

+
More responsive embeddings

The state remains sensitive to current behavior instead of depending only on slower model-driven dynamics.

+
Direct self-correction

Fresh observations actively correct the predicted state, creating a tighter prediction-correction loop.

Product Framing

What this project is built to solve

Behavior changes quickly

Static recommenders can lag behind new signals. This system is designed for environments where user intent shifts quickly and recommendations need to adapt in near real time.

Prediction alone is not enough

A graph model can learn structure, but fresh user activity still needs a fast local update. The filtering stage helps close that gap between global learning and immediate response.

Explicit state evolution

The method treats node embeddings as a time-evolving state, making the interaction between model prediction and observation-driven correction explicit.

System Design

The recommendation engine in four stages

01

Temporal event intake

User-item interactions are sorted over time, grouped into buckets, and treated as an online stream rather than a static batch dataset.

02

Graph-based prediction

A lightweight GNN produces a prior state over node embeddings, using recency-aware edge weights to preserve structural and temporal signal.

03

Filtering update

After new events arrive, the model performs a local adjustment step that reinforces the observed item and pushes away top competing alternatives.

04

Online ranking optimization

A BPR-style objective trains the model continuously on recent interactions, making the system suitable for adaptive ranking experiments.

Application Example

Streaming recommendation on a dynamic user-item graph

Problem setting

The pipeline is applied to a time-evolving Reddit interaction graph from the JODIE benchmark, where user-item events arrive sequentially and embeddings must be updated without leaking future information into earlier predictions.

Graph construction

The implementation first extracts the largest connected component, sorts all interactions in ascending time order, and treats the remaining graph as the main interaction system for message passing and online updates.

Time-step definition

Instead of using every second as a separate update, interactions are grouped into one-minute buckets. The original timestamps are recorded in seconds, but second-level updates are too sparse for stable learning. Using one-minute steps keeps enough meaningful interactions inside each bucket while still preserving temporal resolution.

Prediction-Correction Loop

How the loop behaves over time

Each update starts from the current embedding state, uses the GNN to produce a structured prior, and then applies a direct interaction-driven correction from the newest observations.

The corrected state becomes the starting point for the next step, which makes the pipeline both model-aware and observation-responsive instead of relying on a purely learned update rule.

Advantages Over Purely Model-Driven Recommenders

Model-driven prediction

The GNN provides a prior over the embedding state rather than handling every update alone.

Observation-driven correction

New interactions directly modify the current state through filtering instead of waiting for retraining.

Explicit control of dynamics

The update weights balance prior state, learned prediction, and new observations in a controlled loop.

Separation of roles

The model captures graph structure while the filtering step handles immediate adaptation in streaming settings.

Smarter temporal discretization

Second-level timestamps are aggregated into one-minute buckets so each update step contains enough signal to be informative without becoming too coarse.

Method Properties

Core characteristics of the prediction-filtering framework

Hybrid prediction + correction

The project does more than generate scores. It introduces an adaptive correction step inspired by filtering and data assimilation ideas.

Streaming mindset

The architecture is organized around temporal events and incremental updates, which makes it a stronger fit for modern recommendation problems than a purely static pipeline.

Evaluation beyond one metric

The system design already considers `Hit@K`, `MRR`, and `NDCG`, showing good modeling discipline and awareness of ranking tradeoffs.

Flexible integration

The framework is flexible and can be embedded with temporal GNN architectures or reinforcement learning algorithms, making it adaptable to a broader range of dynamic recommendation settings.
